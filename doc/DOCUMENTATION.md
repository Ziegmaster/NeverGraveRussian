# Документация мода Never Grave Russian

## Обзор
Мод **Never Grave Russian** предназначен для локализации игры *Never Grave* на русский язык. Помимо простой замены текста, мод динамически адаптирует пользовательский интерфейс (UI) игры, чтобы более длинные русские слова корректно отображались без обрезания и наложений.

---

## 1. Система перевода

### 1.1. Загрузка и хранение
- Мод загружает переводы из файла `Mods/NeverGraveRussian.json` при запуске игры.
- Переводы хранятся в оперативной памяти в виде `Dictionary<string, string>`, что обеспечивает мгновенный доступ (`O(1)`) при рендеринге текста.

### 1.2. Обработка динамического текста
Игровой текст часто содержит динамические элементы (числа, HTML-теги, плейсхолдеры). Чтобы не создавать тысячи уникальных строк для каждого числа (например, "Урон: 10", "Урон: 15"), мод использует систему нормализации:
1. Регулярное выражение `SmartRegex` находит все числа, теги `<color=...>` и плейсхолдеры `{0}`.
2. Эти элементы временно извлекаются из строки и заменяются на индексы (например, `{0}`, `{1}`).
3. Мод ищет нормализованную строку в словаре.
4. Если перевод найден, извлеченные динамические элементы вставляются обратно в переведенную строку.

### 1.3. Поддержка Wildcard (шаблонов)
Для текстов, которые сложно нормализовать стандартным способом, мод поддерживает специальный тег `{WILDCARD}` в ключах JSON.
- При загрузке словаря ключи с `{WILDCARD}` компилируются в регулярные выражения (`Regex`).
- Если точное совпадение не найдено, мод проверяет строку по всем скомпилированным шаблонам.
- Совпавшие части строки (wildcards) переносятся в переведенную строку.

### 1.4. Сбор непереведенного текста
Если текст содержит английские буквы и не найден в словаре, мод автоматически сохраняет его в файл `Mods/NeverGraveUntranslated.json`.
- Для предотвращения спама и падения производительности, за одну игровую сессию сохраняется не более 20 новых строк.

## 2. Адаптация пользовательского интерфейса (UI)

Русский текст часто длиннее английского, что ломает оригинальную верстку игры. Мод перехватывает методы установки текста (`text` setter и `SetText`) у компонентов `TMPro.TMP_Text` и применяет специфичные патчи.

### 2.1. Идентификация элементов
Для применения патчей к конкретным окнам мод вычисляет полный путь элемента в иерархии Unity (например, `MvUI_Title>Canvas>Menu_Top>Info_Version>Txt_Version`).

### 2.2. Статические корректировки (Hardcoded Patches)
Некоторые элементы интерфейса имеют фиксированные размеры, которые мод принудительно изменяет один раз при их появлении:
- **Версия игры:** Добавляется надпись "ПЕРЕВОД: ZIEGMASTER". Якорь (pivot) текста смещается вниз, чтобы при добавлении новых строк текст рос вверх, а не перекрывал другие элементы.
- **Меню настроек (`MvUI_SettingSelect`):** Блок расширяется до 60% ширины экрана и центрируется.
- **Заголовки экипировки (`Txt_EqNumTitle`):** Сдвигаются влево и расширяются, чтобы вместить длинные названия.

### 2.3. Динамическое масштабирование (Dynamic Layouts)
Для всплывающих окон (попапов) и диалогов мод рассчитывает размеры динамически, основываясь на длине текста.

**Механизм работы:**
1. **Отключение переноса:** Временно отключается `enableWordWrapping`, чтобы узнать идеальную ширину текста (`preferredWidth`) в одну строку.
2. **Расчет ширины:** 
   - Если текст короткий, ширина устанавливается по тексту.
   - Если текст длинный, устанавливается максимальная ширина (например, `600f` для диалогов), а текст принудительно переносится на новые строки.
3. **Применение LayoutElement:** На текстовый компонент вешается `LayoutElement`, которому задается вычисленная `preferredWidth`.
4. **Принудительный ребилд (Force Rebuild):** Вызывается `LayoutRebuilder.ForceRebuildLayoutImmediate()`. Это критически важный шаг, который заставляет Unity пересчитать высоту текста с учетом новой ширины *в том же кадре*. Это устраняет баг с задержкой в 1 кадр (когда фон сначала слишком большой, а потом сжимается).
5. **Подгонка фона:** На родительский элемент (фон) вешается `ContentSizeFitter`, который автоматически обтягивает текст по высоте и ширине с учетом заданных отступов (padding).

**Элементы с динамическим масштабированием:**
- Диалоги (`MvHud_TalkAuto`)
- Окна взаимодействия (`Bg_Interact`)
- Всплывающие окна разблокировки (`MvUI_ElemUnlock`)
- Элементы лотереи/гачи (`MvUI_ElemLot`)

## 3. Система обновлений
Мод асинхронно обращается к удаленному серверу (GitHub) для проверки актуальной версии перевода. Если доступна новая версия, на главном экране игры появляется зеленая надпись "ДОСТУПНО ОБНОВЛЕНИЕ".

---

## 4. Оптимизация и производительность
- **In-Memory Cache:** Все переводы хранятся в оперативной памяти.
- **Кэширование патчей UI:** Чтобы не применять тяжелые операции (добавление компонентов, пересчет Layout) каждый кадр, мод сохраняет `InstanceID` уже обработанных объектов в `HashSet<int> patchedComponents`. Если объект уже пропатчен, логика мгновенно пропускается.
- **Отсутствие логгирования в Release:** В финальной версии кода убраны все вызовы `MelonLogger.Msg` и рекурсивные обходы иерархии для логгирования, что исключает лишнюю нагрузку на Garbage Collector.
